# 贡献指南

感谢你关注勤记。提交问题或改动前，请先确认内容不包含姓名、地址、证件号码、工资单、雇主资料、备份文件或其他个人信息。

## 报告问题

请在 GitHub Issues 中说明：

- 设备型号、iOS 与 Xcode 版本；
- App 使用的语言；
- 可重复的问题步骤、实际结果和预期结果；
- 如需截图，请先遮盖所有个人与雇主信息。

安全或隐私漏洞请按照 [安全说明](SECURITY.md) 私下报告，不要创建公开 Issue。

## 提交代码

1. 从 `main` 创建功能分支，保持每次改动范围清晰。
2. 不提交签名证书、开发 Team、真实 Bundle ID、用户数据或生成文件。
3. 界面文字需要同时提供简体中文、日文和英文，并避免直接显示本地化键名。
4. 工时和工资逻辑放在计算层，货币继续使用 `Decimal`，不要在界面中复制业务规则。
5. 为行为变化补充单元测试或 UI 测试，并在提交前运行相关测试。
6. Pull Request 中说明改动目的、验证方式、隐私影响和已知限制。

提交贡献即表示你有权提交相关内容，并同意项目维护者在本仓库现行许可条款下使用该贡献。维护者可能要求在合并前补充单独授权。

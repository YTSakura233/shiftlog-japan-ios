# 安全说明

## 报告漏洞

请不要在公开 Issue、截图、日志或示例备份中提交真实个人数据。发现可能导致数据泄露、权限绕过、文件暴露或错误恢复的问题时，请使用 GitHub 仓库的 **Security → Report a vulnerability** 私下报告。

报告中可以包含：

- 受影响版本与设备环境；
- 最小复现步骤；
- 可能的影响；
- 已去除个人信息的截图或示例数据。

维护者会先确认报告，再根据风险安排修复和公开说明。请在修复发布前避免公开漏洞细节。

## 支持范围

当前优先处理 `main` 分支和最新发布版本中的安全问题。旧版本可能只提供升级建议。

本项目不会要求你发送真实工资单、证件、雇主资料、JSON 备份、签名证书或 Apple 开发者账号信息。
